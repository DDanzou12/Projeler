JWT ile API Güvenliği

Bu çalışmada ASP.NET Core Web API projesinde, kullanıcı giriş yaptıktan sonra istemciye JSON Web Token (JWT) verilerek sonraki isteklerde yetkilendirme yapılması amaçlanmıştır. Böylece API uç noktaları daha güvenli hale getirilmiştir.

1. Projenin Amacı

Bu projenin temel amacı, kullanıcı doğrulama ve yetkilendirme işlemlerini token tabanlı bir yapı ile güvenli şekilde gerçekleştirmektir. Kullanıcı sisteme giriş yaptıktan sonra sunucu tarafından bir JWT oluşturulur. Kullanıcı daha sonra bu token'ı her istekte göndererek korumalı API endpointlerine erişebilir.

2. Kullanılan Teknolojiler

ASP.NET Core Web API

API geliştirme ve endpoint yönetimi için kullanılmıştır.

JWT

Kullanıcı doğrulama sonrası token üretmek için kullanılmıştır.

Authentication / Authorization

Kullanıcı kimlik kontrolü ve yetki denetimi için kullanılmıştır.

Entity Framework Core

Kullanıcı verilerini veritabanında saklamak için tercih edilmiştir.

3. JWT Çalışma Mantığı

1. Kullanıcı giriş yapar
2. Bilgiler sunucuda kontrol edilir
3. Geçerliyse JWT üretilir
4. Token istemciye gönderilir
5. Sonraki isteklerde token header'a eklenir
6. API token'ı doğrular

4. Yapılan İşlemler

  • Kullanıcı giriş işlemi için bir login endpointi oluşturuldu.
  • Doğru kullanıcı adı ve şifre girildiğinde JWT üretildi.
  • appsettings.json içinde gizli anahtar, issuer ve audience bilgileri tanımlandı.
  • Program.cs dosyasında JWT doğrulama ayarları yapıldı.
  • [Authorize] attribute'u ile korumalı endpointler oluşturuldu.
  • İsteklerde Authorization: Bearer TOKEN yapısı kullanıldı.

5. appsettings.json Örneği

{
  "Jwt": {
    "Key": "BuBenimGizliAnahtarim12345",
    "Issuer": "JwtDemoApi",
    "Audience": "JwtDemoClient"
  }
}

6. Program.cs Yapılandırması

builder.Services.AddAuthentication("Bearer")
    .AddJwtBearer("Bearer", options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateIssuer = true,
            ValidateAudience = true,
            ValidateLifetime = true,
            ValidateIssuerSigningKey = true,
            ValidIssuer = builder.Configuration["Jwt:Issuer"],
            ValidAudience = builder.Configuration["Jwt:Audience"],
            IssuerSigningKey = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(builder.Configuration["Jwt:Key"]))
        };
    });

builder.Services.AddAuthorization();

7. Token Oluşturma Mantığı

var claims = new[]
{
    new Claim(ClaimTypes.Name, user.UserName),
    new Claim(ClaimTypes.Role, "User")
};

var key = new SymmetricSecurityKey(
    Encoding.UTF8.GetBytes(_configuration["Jwt:Key"]));

var creds = new SigningCredentials(key, SecurityAlgorithms.HmacSha256);

var token = new JwtSecurityToken(
    issuer: _configuration["Jwt:Issuer"],
    audience: _configuration["Jwt:Audience"],
    claims: claims,
    expires: DateTime.Now.AddMinutes(30),
    signingCredentials: creds);

return new JwtSecurityTokenHandler().WriteToken(token);

8. Korumalı Endpoint Örneği

[Authorize]
[HttpGet("profil")]
public IActionResult GetProfile()
{
    return Ok("Bu alana sadece token sahibi kullanıcı erişebilir.");
}

9. Güvenlik Açısından Sağladığı Faydalar

Oturum Bilgisi Tutulmaz

Sunucu tarafında session tutulmadan güvenli işlem yapılabilir.

Rol Tabanlı Yetki

Token içine rol bilgisi eklenerek farklı yetkiler verilebilir.

Kolay Entegrasyon

Web, mobil ve farklı istemcilerle rahat şekilde kullanılabilir.

10. Sonuç

Sonuç olarak JWT kullanımı, ASP.NET Core Web API projelerinde güvenli ve modern bir kimlik doğrulama yöntemi sunmaktadır. Bu yapı sayesinde kullanıcı giriş yaptıktan sonra her istek için tekrar tekrar giriş yapmasına gerek kalmadan güvenli bir şekilde sistemde işlem yapabilmektedir. Özellikle REST API projelerinde JWT kullanımı oldukça yaygındır ve pratik bir çözümdür.
